Introduction to Brainstorm AI API

Welcome to the Brainstorm AI API documentation. Our API provides powerful AI chatbot capabilities that can be integrated into your applications, websites, and services. With multiple specialized models, you can tailor the AI's behavior to your specific use case.

Brainstorm AI offers natural language understanding and generation capabilities that enable your applications to engage in meaningful conversations with users, answer questions, generate content, process text, and much more.

Note: This documentation covers the Brainstorm AI API v2.5.0. If you're using an older version, some features and endpoints may not be available.

Key Features

  • Multiple Specialized Models - Choose from five different AI models optimized for various use cases
  • RESTful API - Simple HTTP requests with JSON responses
  • Streaming Support - Stream responses for real-time interactions
  • Robust SDKs - Libraries available for major programming languages
  • Fine-tuning - Customize models with your own data
  • Moderation - Built-in content moderation capabilities
  • High Availability - 99.99% uptime guarantee with enterprise plans

Quick Start Guide

Follow these steps to start using the Brainstorm AI API in your application:

1. Sign Up for an API Key

Create an account on the Brainstorm AI Dashboard and generate an API key.

2. Install the SDK (Optional)

While you can use any HTTP client to interact with our API, we recommend using one of our official SDKs:

npm
npm install brainstorm-ai-sdk
pip
pip install brainstorm-ai

3. Make Your First API Call

Here's a simple example of how to use the Chat API:

JavaScript
const Brainstorm = require('brainstorm-ai-sdk');

// Initialize with your API key
const brainstorm = new Brainstorm('your-api-key');

// Send a chat request
async function getResponse() {
  const response = await brainstorm.chat.create({
    model: 'standard',
    messages: [
      { role: 'user', content: 'Hello, how can you help me today?' }
    ]
  });

  console.log(response.choices[0].message.content);
}

getResponse();
Python
import brainstorm_ai as brainstorm

# Initialize with your API key
client = brainstorm.Client(api_key="your-api-key")

# Send a chat request
response = client.chat.create(
    model="standard",
    messages=[
        {"role": "user", "content": "Hello, how can you help me today?"}
    ]
)

print(response.choices[0].message.content)

Authentication

All API requests must include your API key for authentication. You can manage your API keys in the Brainstorm AI Dashboard.

API Key Authentication

Include your API key in the request headers:

HTTP Header
Authorization: Bearer YOUR_API_KEY
Security Warning: Protect your API keys and never expose them in client-side code. Always make API requests from your server.

API Key Rotation

We recommend rotating your API keys periodically for enhanced security. You can generate multiple keys and gradually transition your applications to the new key before deleting the old one.

API Models Overview

Brainstorm AI offers different models optimized for various use cases. Each model has different capabilities, pricing, and performance characteristics.

Brainstorm Standard
General Purpose

Our baseline model offering a good balance of capabilities for general conversational AI applications. Ideal for customer service, simple Q&A, and basic content generation.

Context Window: 8,000 tokens

Max Output: 2,000 tokens

Brainstorm Pro
Enhanced Performance

Advanced model with improved reasoning, context understanding, and response quality. Recommended for complex customer interactions, tutoring, and detailed responses.

Context Window: 16,000 tokens

Max Output: 4,000 tokens

Brainstorm Scientific
Research & Analysis

Specialized for scientific, technical, and analytical domains. Excels at problem-solving, data analysis, and explaining complex concepts in STEM fields.

Context Window: 24,000 tokens

Max Output: 4,000 tokens

Brainstorm Creative
Content Generation

Optimized for creative writing, storytelling, marketing copy, and other content generation tasks. Produces more engaging and diverse content.

Context Window: 16,000 tokens

Max Output: 8,000 tokens

Brainstorm Enterprise
Maximum Capability

Our most powerful model with the best performance across all tasks. Features advanced reasoning, larger context windows, and improved output quality. Includes priority infrastructure for faster response times.

Context Window: 32,000 tokens

Max Output: 8,000 tokens

Chat Endpoint

The Chat API allows you to have interactive conversations with the AI model. It maintains context over multiple messages, enabling more natural and coherent dialogues.

POST https://api.brainstorm.ai/v1/chat

Request Parameters

Parameter Type Required Description
model string Yes The model to use: "standard", "pro", "scientific", "creative", or "enterprise"
messages array Yes An array of message objects representing the conversation history
temperature number No Controls randomness: 0.0-1.0 (default: 0.7)
max_tokens integer No The maximum number of tokens in the response
stream boolean No Whether to stream the response (default: false)

Message Object Format

Parameter Type Required Description
role string Yes Either "system", "user", or "assistant"
content string Yes The content of the message

Example Request

JSON
{
  "model": "standard",
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful customer service assistant."
    },
    {
      "role": "user",
      "content": "I need help tracking my order #12345."
    }
  ],
  "temperature": 0.7,
  "max_tokens": 500
}

Best Practices

Follow these guidelines to get the most out of the Brainstorm AI API:

Prompt Engineering

The quality of your prompts significantly affects the quality of the responses. Here are some tips:

  • Be specific and clear in your instructions.
  • Provide examples of desired outputs when possible.
  • Use system messages to set the behavior and tone of the assistant.
  • Break complex tasks into smaller, manageable steps.
  • Experiment with temperature settings to find the right balance between creativity and determinism.

Optimizing Token Usage

Token optimization can help reduce costs and improve performance:

  • Limit conversation history by summarizing or removing older messages when they're no longer relevant.
  • Set appropriate max_tokens limits to prevent unnecessarily verbose responses.
  • Use the most appropriate model for your task rather than defaulting to the most powerful one.
  • Structure your prompts efficiently to communicate the necessary information with fewer tokens.

Error Handling

Implement robust error handling in your application:

  • Implement retry logic with exponential backoff for rate limit errors or temporary service issues.
  • Validate user inputs
